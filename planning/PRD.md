# PRD - web-cowork-06-ui-3

## 1. 프로젝트 개요
- Objective: 로딩/에러/빈 상태 UI 3종 구현 및 상태 전환 데모 구성
- Brand Tone: 신뢰감 있는 프리미엄
- Target Audience: 온라인 구매 의사가 있는 일반 고객
- Priority: P1
- Deadline: 2026-03-19 23:59 (KST)

## 2. 범위(동결)
- 필수 섹션: `hero`, `product`, `trust`, `cta`
- 필수 상태: `loading`, `error`, `empty`, `success`
- 데모 범위: 전체 상태 일괄 전환 + 섹션 단위 상태 전환 + `error -> retry -> loading -> success` 시나리오
- Core CTA: `지금 시작하기` (성공/유효 상태에서 일관 노출)
- 금지 요소: 과장된 허위 문구

## 3. 공통 UX 원칙
- Loading: 스켈레톤 기반, 섹션 최소 높이 고정, 진행 안내 1문장
- Error: 원인 + 행동(재시도) 구조, 불안 조장/책임 전가 문구 금지
- Empty: 데이터 부재를 명확히 알리고 다음 행동 1개 제시
- Success: 핵심 정보와 `지금 시작하기`를 가장 눈에 띄는 위치에 배치

## 4. 섹션별 상태 카피(기준본)
### 4.1 hero
- loading: `맞춤 제안을 준비하고 있습니다. 잠시만 기다려 주세요.`
- error: `시작 정보를 불러오지 못했습니다. 네트워크를 확인한 뒤 다시 시도해 주세요.`
- empty: `아직 맞춤 데이터가 없습니다. 간단한 설정 후 제안을 확인해 보세요.`
- success CTA: `지금 시작하기`

### 4.2 product
- loading: `상품 정보를 정리하고 있습니다.`
- error: `상품 목록을 가져오지 못했습니다. 잠시 후 다시 시도해 주세요.`
- empty: `현재 조건에 맞는 상품이 없습니다. 필터를 조정해 보세요.`
- success CTA: `지금 시작하기`

### 4.3 trust
- loading: `검증 정보를 확인하고 있습니다.`
- error: `신뢰 정보를 불러오지 못했습니다. 다시 시도해 주세요.`
- empty: `표시할 인증/리뷰가 없습니다. 최신 검증 데이터를 준비 중입니다.`
- success CTA: `지금 시작하기`

### 4.4 cta
- loading: `신청 화면을 준비하고 있습니다.`
- error: `시작 화면 연결에 실패했습니다. 다시 시도해 주세요.`
- empty: `바로 시작 가능한 기본 안내를 제공하고 있습니다.`
- success CTA: `지금 시작하기`

## 5. 제약 관리
- 예산 제약: 24 인시(±10%) 내 설계/구현/QA 완료
- 기간 제약: 2026-03-19 마감 고정, 카피 승인 지연 시 상태 데모 구현을 우선 착수
- 품질 제약:
  - 4개 섹션 x 4개 상태 매트릭스 누락 0건
  - 상태 전환 오작동 0건
  - CTA 문구 불일치 0건
  - 과장/허위 문구 검출 0건

## 6. 콘텐츠 컴플라이언스
- 금지 예시: `업계 1위 보장`, `무조건 최저가`, `100% 성공`
- 수치/우위 표현은 근거 데이터가 있을 때만 허용
- 사용자 불안을 자극하는 표현(예: `지금 안 하면 손해`) 금지

## 7. 완료 기준
- `planning/design_spec.md`와 본 문서에 섹션별 상태 카피와 컴포넌트 배치 규칙이 반영되어야 함
- `hero/product/trust/cta` 전 섹션에서 상태 전환 데모 기준이 일관되어야 함
- QA 테스트 포인트(상태 전환, 재시도, CTA 노출 조건, 금지 문구) 기준 자체 점검 기록이 있어야 함
